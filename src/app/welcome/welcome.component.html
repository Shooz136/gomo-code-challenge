For my implementation of the gomo Code Challenge, I've created an Angular 7 application with a Firebase backend and Material front-end.

The application simulates 'watching' fragments of a video, and then viewing the watched fragments, the coalesced contiguous fragments, the UVT, and the watched fragments plotted on a timeline.

The application should work equally well on a mobile browser as in in a desktop browser.

This application is meant to fulfill the requirements of the challenge in an interesting way, not approximate any real-world, end-user functionality.

To get started:

Choose one of the two users in the side navgiation bar, or use these links: (User 1, User 2).

Once you're on a user page, use the play 'FAB' button in the bottom right corner to add new fragments.

When you've added some fragments, you can click through the tabs to view the resultant data, aggregates, and timeline.

Known Issues / Concerns:

These are just a few of the issues and concerns I've noticed during development. Most of these issues come down to time constraints.

Testing Testing Issues

While I did write a test for the fragment-service and left some tests for the more trivial controllers, I did not fully test the non-trivial controllers - i.e. add-frag-modal, video-view, etc.

In order to test these controllers properly, we would need to stub all of the firebase services.

This is completely doable, but I've left it outside the scope of this project.

Data Validation Data Validation Issues

I left off some of the data validation on the fragment input form due to time constraints. That being said, front-end validation is reasonably trivial to implement, especially with the Angular framework.

Firebase also has a back-end validation framework that will let us fail transactions that don't conform to given criteria; again omitted due to time constraints.

 Timeline Component Timeline Component Issues

The timeline component is far too highly coupled to our domain model - it really just needs start and length percentages, not VideoFragments.

Ideally, we'd probably spin this off into its own module, allowing it to be reused in other projects, and tested without Firebase stubs.

Again, decisions were made for efficiency reasons.

 Fragment Modal Add Fragment Modal Bug

The add-fragment modal will trigger a failure snackbar if a user clicks off of it - this is a bug.

I should've returned more contextual information out of the modal, rather than a simple boolean value.
